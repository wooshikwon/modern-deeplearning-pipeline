## 핵심 발견

### 문서 의도 우선 — 코드를 문서에 맞춰야 하는 곳

#### [Critical] 런타임 버그 — 코드 직접 검증 완료

**1. `cli/train.py:89` — `create_job()` 필수 인자 누락** ✅ 확인

```python
job_id = manager.create_job(executor="local")
```

`JobManager.create_job(self, job_id: str, executor: str, ...)` — `job_id`가 필수 위치 인자인데 누락. 런타임 `TypeError` 확정.

**2. `cli/inference.py:127` — `compute_baseline()` 파라미터명 불일치** ✅ 확인

```python
current = compute_baseline(model=model, dataloader=test_loader)
```

`compute_baseline(train_dataloader, model, config, *, max_batches)` — `dataloader`는 존재하지 않는 파라미터. 런타임 `TypeError` 확정.

**3. `data/dataloader.py:90-111` — `drop_last` 이중 전달** ✅ 확인

`loader_kwargs = dict(loader_config)` 에서 `drop_last`를 pop하지 않은 채 `_make_loader`의 명시적 `drop_last` 파라미터와 `**loader_kwargs` 모두로 `DataLoader`에 전달. `DataloaderSpec`이 항상 `drop_last`를 포함하므로 **정상 경로에서 항상 `TypeError: got multiple values for keyword argument 'drop_last'` 발생**.

**4. `models/adapters/qlora.py:81-93` — `modules_to_save`가 `from_pretrained`에 불필요하게 전달** ⚠️ 심각도 하향

```python
model = model_cls.from_pretrained(..., **kwargs)  # modules_to_save가 kwargs에 포함
modules_to_save = kwargs.pop("modules_to_save", None)  # pop은 정상 작동
```

> **1차 분석 오류 수정**: sub-agent는 "pop이 from_pretrained 이후라 실패한다"고 보고했으나, **Python의 `**kwargs` 언팩은 원본 dict를 수정하지 않으므로 pop은 정상 작동한다**. 실제 문제는 `modules_to_save`가 `from_pretrained`에 불필요하게 전달되는 것으로, HuggingFace `from_pretrained`가 미지원 kwarg를 `__init__`에 전달하여 경고 또는 무시할 수 있다. **크래시 아닌 설계 결함**.

**5. `training/trainer.py:308` — `_log_to_mlflow`가 MLflow context 밖에서 호출** ⚠️ 심각도 하향

```python
with mlflow_ctx:          # mlflow.start_run() context
    try:
        for epoch in ...:
            ...              # _mlflow_log_metric 호출 (정상)
        ...
    finally:
        ...
                              # with 블록 종료 → mlflow.end_run()
training_duration = ...
self._log_to_mlflow(...)     # active run 종료 후 호출
```

`_log_to_mlflow`에 `try/except Exception`이 있어 **크래시하지 않으나**, 최종 params/metrics/config가 MLflow에 기록되지 않는 **기능적 버그**. 학습 중 step별 로깅은 정상 동작.

### 오탐지 — 1차 분석에서 제거된 항목

#### ❌ `qlora.py:93` "pop이 from_pretrained 이후라 실패"

**원인**: Python `**kwargs` 언팩은 dict를 복사하여 전달하므로 원본 dict는 수정되지 않음. `kwargs.pop("modules_to_save")` 은 언팩 후에도 정상 작동. 실제 문제는 불필요한 kwarg가 `from_pretrained`에 전달되는 것으로 **심각도 재분류** (P0→P3).

#### ❌ `trainer.py:295` "baseline_info UnboundLocalError 가능성"

**원인**: `baseline_info` 할당은 `try` 블록 마지막에 위치. 이전에 예외가 발생하면 `finally` 실행 후 예외가 전파되어 `with` 블록을 벗어나므로, `baseline_info`를 참조하는 line 317은 **도달 불가**. UnboundLocalError는 발생할 수 없음.

#### ❌ `pretrained.py:68` "rstrip이 잘못된 결과 생성"

**원인**: sub-agent가 `rstrip(":/")` 동작을 오해. `rstrip`은 **문자 집합** `{:, /}`에서 오른쪽 문자를 제거하는 것이지 부분 문자열을 제거하는 게 아님. `"local://"` → `"local"` 정상 동작 확인. `removesuffix("://")`가 의도를 더 명확히 표현하지만 기능 버그는 아님.

#### ❌ `_select_collator`에 token_classification 분기 없음 → "명확한 버그"

**원인**: `LABEL_ALIGN` strategy는 `else` 분기에서 `DataCollatorWithPadding`으로 처리됨. NER 등에서 `DataCollatorForTokenClassification`이 더 적합할 수 있으나, `DataCollatorWithPadding`도 동작하며 명확한 크래시 버그가 아님. **논의 필요**로 재분류.

### 논의 필요

**1. `DataSpec` flat vs `_component_` 패턴** — 현재 HuggingFace 전용 flat 구조 vs 문서의 `_component_` 기반 추상화. 통일 방향 결정 필요.

**2. `validate_task_fields` 미호출** (`task_taxonomy.py:81-108`) — 정의되어 있으나 어떤 validator에서도 미사용. 통합 또는 제거.

**3. `TASK_PRESETS` vs `TASK_HEAD_COMPAT` 이원 관리** — 8개 vs 9개 task. 단일 소스 부재.

**4. `prefix_tuning` 지원 범위** — 문서 명시이나 구현 없음. 계획 반영인지 오기인지.

**5. CatalogValidator 반환 타입 불일치** — `list[str]` vs `ValidationResult`. 통일 여부.

**6. `deepspeed_zero3` alias** — `deepspeed`와 동일 클래스 매핑. ZeRO-3 + offload 설정 주입 경로 없음.

**7. `best_metrics` 덮어쓰기** (`trainer.py:287`) — `update(val_metrics)`가 항상 최신 val 결과로 교체. "best" 의미와 불일치 가능.

**8. `loader.py` 사용 경로 불분명** — `create_dataloaders()`와 독립적인 transform 경로. 통합 여부.

**9. `tokenizer.py:138` target 컬럼 하드코딩** — `"summary"` 하드코딩.

**10. `max_epochs = 10_000` 매직 넘버** (`trainer.py:261`) — `max_steps` 모드 묵시적 상한.

**11. `checkpoint.on_batch_end`의 step 기준** (`checkpoint.py:173`) — 에폭 내 배치 인덱스 사용. global_step이 `kwargs`로 전달되지만 미사용. `storage.checkpoint_every_n_steps`와 연결 없음.

**12. DeepSpeed `gradient_clipping: 1.0` 하드코딩** — `training.gradient_clip_max_norm` 무시 가능성.

**13. `serving: None` 기본값** (`schema.py:210`) — 문서는 `serving.backend` 기본 `"torchserve"`. None vs 기본 인스턴스.

**14. `head_builtin: true` + `default_head` 공존** — 우선순위 정책 불명확.

**15. `DualEncoderHead.forward_pair`가 ABC 밖** — BaseHead ABC 확장 또는 별도 인터페이스 필요 여부.
